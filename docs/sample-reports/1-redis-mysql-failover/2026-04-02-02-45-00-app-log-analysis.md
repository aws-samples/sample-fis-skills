## Recommendations

1. **catalog 应用 MySQL 连接池优化**
   - **Impact:** RDS failover 期间 10 秒连接超时导致请求阻塞
   - **Recommendation:** 配置 Go SQL 驱动参数 `ConnMaxLifetime=5m`、`ConnMaxIdleTime=1m`，添加连接验证 (`ping` before use)，减少 dial timeout 到 3 秒以加快故障检测

2. **UI 层添加服务降级策略**
   - **Impact:** catalog 不可用时 UI 直接返回 500，影响所有用户
   - **Recommendation:** 实现 circuit breaker 模式，当 catalog 服务不可用时返回缓存数据或友好的降级页面

3. **添加 MySQL DNS 缓存 TTL 配置**
   - **Impact:** DNS 切换后应用可能仍使用旧 IP 建立连接
   - **Recommendation:** 确保 JVM/Go runtime 的 DNS 缓存 TTL ≤ 5 秒 (`networkaddress.cache.ttl=5`)

4. **checkout/Redis 集群模式配置可作为最佳实践推广**
   - **Impact:** checkout 在整个 AZ 电源中断期间零错误
   - **Recommendation:** 将 Redis 集群模式 + 跨 AZ 副本的配置作为团队最佳实践，其他服务参考此架构
